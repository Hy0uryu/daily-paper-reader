## 摘要
当用户询问“你把我的钥匙放哪了？”或“任务为什么失败了？”时，机器人必须能够用语言表达其过去的经历。然而，从持续的多模态感知中维持终身情景记忆（EM）会迅速超出存储限制，并使实时查询变得不切实际，因此需要根据用户对相关性的理解进行选择性遗忘。我们提出了 H$^2$-EMV，这是一个使类人机器人能够通过用户交互学习该记住什么的框架。我们的方法增量地构建分层情景记忆，利用基于语言模型且以学习到的自然语言规则为条件的相关性估计进行选择性遗忘，并根据用户对遗忘细节的反馈更新这些规则。在模拟家庭任务和来自 ARMAR-7 的 20.5 小时真实世界录像上的评估表明，H$^2$-EMV 在保持问答准确性的同时，将记忆大小减少了 45%，查询时的计算量减少了 35%。至关重要的是，性能随时间而提高——通过适应用户特定的优先级，第二轮查询的准确率提高了 70%——这证明了学习遗忘能够为长期人机协作实现可扩展且个性化的情景记忆。

## Abstract
Robots must verbalize their past experiences when users ask "Where did you put my keys?" or "Why did the task fail?" Yet maintaining life-long episodic memory (EM) from continuous multimodal perception quickly exceeds storage limits and makes real-time query impractical, calling for selective forgetting that adapts to users' notions of relevance. We present H$^2$-EMV, a framework enabling humanoids to learn what to remember through user interaction. Our approach incrementally constructs hierarchical EM, selectively forgets using language-model-based relevance estimation conditioned on learned natural-language rules, and updates these rules given user feedback about forgotten details. Evaluations on simulated household tasks and 20.5-hour-long real-world recordings from ARMAR-7 demonstrate that H$^2$-EMV maintains question-answering accuracy while reducing memory size by 45% and query-time compute by 35%. Critically, performance improves over time - accuracy increases 70% in second-round queries by adapting to user-specific priorities - demonstrating that learned forgetting enables scalable, personalized EM for long-term human-robot collaboration.

#### 1. 核心问题与整体含义
在机器人的终身部署（Lifelong Deployment）中，机器人需要通过情景记忆（Episodic Memory, EM）来记录并回答有关过去经历的问题（如“我的钥匙在哪？”）。然而，持续的多模态感知会产生海量数据，导致存储空间迅速耗尽且实时查询效率低下。
**研究动机**：人类通过“选择性遗忘”来管理记忆，但对于机器人而言，“什么是重要的”并没有统一标准。因此，论文旨在开发一种能够**增量构建记忆、根据用户偏好学习遗忘规则、并动态优化存储与查询效率**的框架。

#### 2. 方法论：H$^2$-EMV 框架
论文提出了 **H$^2$-EMV**（Hierarchical Episodic Memory with Forgetting and Learning），其核心由三个关键组件构成：
*   **在线分层历史树构建（Online History Tree Construction）**：
    *   将原始观测流增量地转化为分层树状结构。底层是场景图，中层是事件和目标摘要，高层是递归生成的活动摘要。
    *   **时间感知算法**：通过 LLM 进行语义分组，并结合时间聚类算法防止将时间跨度过大的无关事件合并。
*   **基于衰减的遗忘机制（Decay-based Forgetting）**：
    *   每个记忆节点都有初始寿命。当寿命到期时，系统调用 LLM 评估该节点的相关性（Relevance Factor $\alpha$）。
    *   如果相关性高，则延长寿命；如果相关性低，则将其“遗忘”（用占位符替换，仅保留极简摘要以维持树结构完整性）。
*   **反馈驱动的相关性规则学习（Feedback-based Relevance Learning）**：
    *   将相关性定义为一组**自然语言规则**。
    *   当用户询问已被遗忘的信息并提供反馈（如“你应该永远记住我移动钥匙的时间”）时，LLM 会更新这些规则。这些规则随后指导未来的摘要生成和遗忘决策。

*   **效率提升**：H$^2$-EMV 将记忆存储大小减少了 **45%**，并将查询时的计算开销（Token 消耗）降低了 **35%**。

*   **层次结构的必要性**：实验表明，如果没有分层结构（Flat 模式），随着历史增长，计算成本会爆炸式增长，且遗忘机制难以在无上下文的情况下准确评估相关性。

#### 7. 优点
*   **个性化适应**：通过自然语言规则学习用户特定的偏好，解决了“相关性”难以预定义的问题。
*   **可解释性强**：遗忘规则以自然语言形式存储，用户可以轻松查看、理解和手动修正机器人的记忆逻辑。
*   **系统完整性**：不仅提出了算法，还在真实的人形机器人（ARMAR-7）上实现了闭环部署，验证了异步处理和夜间遗忘等工程实践。

#### 8. 不足与局限
*   **推理延迟**：依赖大型 LLM 进行增量摘要和相关性评估，虽然是异步的，但在信息密集时仍存在分钟级的处理延迟。
*   **规则冲突与幻觉**：使用自然语言修改规则可能导致 LLM 产生幻觉或误删规则；在多用户场景下，不同用户的相关性定义可能存在冲突。
*   **检索错误**：定性分析显示，部分失败案例并非因为信息被遗忘，而是因为 LLM 代理在复杂的树结构中未能正确检索到现有信息。
*   **多模态融合有限**：目前主要依赖文本化的场景描述，对视觉细节（如物体的具体外观变化）的保留和遗忘尚有提升空间。
